# Evolution Theme

当前主题：**让 Agent 更能干、更聪明、能做更多事**

## 能力提升方向

### Coding 更强
- 更智能的代码重构建议（不只是改名，而是识别设计模式缺陷）
- Bug 定位工具 — 给一个报错，自动追踪调用链定位根因
- 代码审查工具 — 不只是 lint，而是理解业务逻辑层面的问题
- Git 智能操作 — 自动生成有意义的 commit message、PR 摘要、changelog

### 新领域拓展
- 数据分析能力 — 读 CSV/Excel、画图表、做统计摘要
- 文档处理 — 读 PDF/Word、提取结构化信息、生成报告
- 办公自动化 — 邮件草稿、会议纪要整理、日程提醒
- 前端开发 — 理解 React/Vue 组件结构、生成 UI 代码

### Agent 更聪明
- 任务失败后自动复盘，总结经验存入 memory，下次类似任务时检索
- 根据历史成功/失败模式自动调整策略（工具选择、重试方式）
- Agent 间能传递"经验"，不是每次从零开始

### 多 Agent 协作新模式
- 辩论模式 — 技术决策时两个 Agent 各持一方，帮用户看清 trade-off
- 红队/蓝队 — 一个 Agent 写代码，另一个专门找漏洞
- 竞速模式 — 同一任务多种策略，选最优方案

### 分析洞察能力
- 项目健康度扫描 — 复杂度趋势、技术债分布、测试覆盖盲区
- 依赖分析 — 找出过时依赖、安全漏洞、循环引用
- 代码库 "X光" — 可视化模块关系、热点区域、变更频率

### Skills 扩展
- 更多专业领域 skill（DevOps、数据库设计、API 设计、前端最佳实践）
- 现有 skill 增强 — 例如 systematic-debugging 加入更多实战模式
- 领域知识 skill — 例如 "Python 性能优化常见模式"、"并发编程陷阱"

## 避免方向
- 纯内部重构（用户感知不到）
- 已有功能的简单 wrapper
- 又一个 logging/monitoring middleware




# Evolution Theme

当前主题：**让 Agent 更能干，能做更多种类的事**

## 思考角度

- 用户拿这个框架做什么事情时会觉得"不够用"？
- 有哪些领域的任务，现在的 Agent 完全帮不上忙？
- 多个 Agent 之间的协作方式，是不是只有"分任务-做-汇报"这一种？还能怎么玩？
- Agent 做完一个任务后，学到的东西能留下来吗？下次能更聪明吗？
- 除了写代码，开发者日常还需要什么？

以下仅为示例，帮助打开思路，**不是必须实现的清单**：
- 例：用户让 Agent 分析一个 CSV 文件，但现在没有数据处理能力
- 例：两个 Agent 对同一个技术问题给出不同方案，用户从中选择
- 例：Agent 做完任务后写一份经验总结，下次类似任务时能检索到
- 例：用户想了解项目的依赖关系或代码健康度，但现在无从下手

## 避免
- 纯内部重构（用户感知不到变化）
- 已有功能的简单 wrapper
- 标准的又一个 middleware
